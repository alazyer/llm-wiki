# `CLAUDE.md` Setup: 21 Things Users Forget

## Summary
`CLAUDE.md` and related memory files are a persistent instruction layer. They save time by making communication preferences, project rules, scope boundaries, and correction history available before each task starts.

## Key Takeaways
- Persistent instructions beat repeating the same prompt setup in every chat.
- Communication rules can define brevity, uncertainty handling, formatting, and no-filler preferences.
- Behavior rules should include scope control, destructive-action confirmation, and summary requirements.
- Context rules should capture user, project, tech stack, style, and domain constraints.
- Memory files such as `MEMORY.md`, session summaries, and `ERRORS.md` preserve lessons across runs.
- Developer guardrails prevent agents from drifting into unwanted tools, stacks, or risky operations.
- Keep instructions compact; bloated instruction files become another context-rot source.
