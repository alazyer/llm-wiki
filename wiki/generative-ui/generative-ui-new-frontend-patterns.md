# Generative UI Is the New Frontend

## Summary
Generative UI lets agents render interfaces dynamically through protocols and schemas rather than only static screens. Production systems need to choose between controlled components, declarative schemas, and open-ended generated UI based on brand, safety, flexibility, and cost.

## Key Takeaways
- The emerging stack includes MCP for tools, A2A for agent communication, AG-UI for user streams, and A2UI for schema-driven UI.
- Controlled UI gives pixel-level control but can inflate tool and token complexity.
- Declarative UI scales through schemas and catalogs; it is a strong default for production agent apps.
- Open-ended HTML or MCP apps work for disposable or exploratory interfaces but are harder to govern.
- Important flows may graduate from declarative to controlled components.
- Generative UI is still frontend engineering: accessibility, latency, safety, and state management remain real.
