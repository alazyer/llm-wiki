# Loop Engineering: Addy Osmani Framing

## Summary
Loop engineering is the layer above prompt engineering and harness engineering: instead of asking an agent to do one thing, design a bounded system that repeatedly finds work, runs an agent, verifies output, and preserves state. The engineer’s job moves toward building the loop while staying accountable for comprehension and review.

## Key Takeaways
- Loop engineering combines automations, worktrees, skills, connectors, sub-agents, and persistent state.
- The loop is only as useful as its verifier; otherwise it becomes faster self-deception.
- Worktrees and one-writer discipline keep parallel agents from trampling each other.
- Skills encode repeatable intent so each run does not rediscover project conventions.
- Connectors let loops act in real systems, but they also widen blast radius.
- Human verification still matters; loops amplify engineers, not replace engineering judgment.
- Main risks are comprehension debt, cognitive surrender, and rubber-stamping generated code.
