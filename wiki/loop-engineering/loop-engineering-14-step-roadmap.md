# Loop Engineering: 14-Step Roadmap

## Summary
The roadmap moves a user from one-off prompting to loop design by asking whether a task is repeated, machine-checkable, end-to-end executable, and worth automating. It then layers in minimum viable loop pieces before adding scale.

## Key Takeaways
- Build loops only for repeated work with objective verification and clear done states.
- Minimum viable loop: automation, skill, state file, and verification gate.
- Add worktrees, connectors, and sub-agents only after the small loop is useful.
- Economics should be measured as cost per accepted change, not cost per model call.
- Common failures include lazy looping, drift, security tax, and unbounded tool access.
- The roadmap’s final warning is to stay the engineer: understand, verify, and own the result.
