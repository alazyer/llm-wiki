# 30 Days Running a 24/7 Agent Team

## Summary
The first month of a persistent agent team is less like installing software and more like onboarding employees. Early output is often mediocre; improvement compounds only when corrections become durable files, rules, skills, and memory.

## Key Takeaways
- Week one can be net negative because agents lack local context and judgment.
- Specific corrections beat motivational prompting.
- Feedback must move from chat into files or it will not compound.
- Scheduler failures, context bloat, and false health signals are predictable operational problems.
- The progression is usually: mediocre output, targeted competence, then compounding returns.
- Start with one agent and one job for 30 days before declaring the approach failed or scaling it.
