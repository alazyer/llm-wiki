# Hermes vs OpenClaw Learning Loops

## Summary
OpenClaw and Hermes represent two improvement-loop styles. OpenClaw relies on human-owned correction and memory files; Hermes aims to turn completed work into reusable skills and playbooks. Both still require verification.

## Key Takeaways
- The key question is who owns the improvement loop: human correction or generated reusable procedures.
- OpenClaw is predictable when humans curate memory and operating files.
- Hermes can capture troubleshooting and repeatable procedures as skills.
- Auto-generated skills are not automatically correct; they need tests and review.
- Day-30 behavior matters more than day-one demo quality.
- A useful stack may combine OpenClaw’s explicit memory with Hermes-style procedure extraction.
