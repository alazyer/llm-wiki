# Taste Index with Hermes and GBrain

## Summary
A Taste Index is a selective memory layer for judgment, not a larger archive. Hermes coordinates work while GBrain stores durable knowledge, but the valuable capture is the explicit reason something matters and how it should shape future agent behavior.

## Key Takeaways
- More memory does not equal better judgment; agents need a selective record of taste.
- A Taste Index stores the artifact plus what the user liked, why it matters, where it should influence future work, and what not to copy blindly.
- The core ingestion rule is “no signal, no storage”; explicit phrases like “save this” or “gbrain this” create taste events.
- Useful captures influence future writing, product, research, design, investing, or agent behavior.
- The two most important fields are “What I liked” and “Why it’s useful.”
- Maintenance should audit existing captures without creating a second ingestion path.
- Twenty strong captures beat thousands of weak notes because taste gains power through selectivity.
