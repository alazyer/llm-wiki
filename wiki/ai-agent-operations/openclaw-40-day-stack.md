# OpenClaw 40-Day Agent Stack

## Summary
The 40-day OpenClaw stack organizes agents into identity, operations, and knowledge layers. It treats files as the integration layer: identities tell agents who they are, operations tell them how to run, and knowledge tells them what the team has learned.

## Key Takeaways
- Identity layer: `SOUL.md`, `IDENTITY.md`, `USER.md`, and role-specific conventions.
- Operations layer: `AGENTS.md`, `HEARTBEAT.md`, schedules, status files, and handoff rules.
- Knowledge layer: `MEMORY.md`, daily logs, shared context, thesis files, and feedback logs.
- One-writer/many-readers reduces conflicts in shared files.
- Shared context lets a correction from one role shape every future role.
- Memory must be compacted; more text is not automatically more intelligence.
- Adoption works best week by week: identity first, then operations, then shared memory.
