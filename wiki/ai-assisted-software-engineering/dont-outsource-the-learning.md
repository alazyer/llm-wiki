# Don’t Outsource the Learning

## Summary
AI can help ship work while quietly weakening the user’s mental model. The antidote is active learning: form hypotheses, ask for explanations, reconstruct solutions, and review AI output like a junior teammate’s pull request.

## Key Takeaways
- Shipping a task is not the same as understanding the system.
- Passive copy-paste use can reduce retention and comprehension.
- Delegate boilerplate, but personally understand architecture, trade-offs, and failure modes.
- Ask for concepts before code when learning a new area.
- Make predictions before asking AI; compare the answer against your model.
- Use AI as a tutor, reviewer, and sparring partner, not only a completion engine.
- Treat generated code as a PR that needs explanation, tests, and ownership.
